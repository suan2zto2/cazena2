    2~5단계 구현 기획 문서 계획

    작성일: 2026-05-07
    대상 구현 기간: 05/25 – 07/03 (2~5단계)
    기준 일정: docs/schedule.md 2~5단계

    1. 기존 문서에서 이미 정의된 내용

    docs/systems/overview.md — 카드·전투·적 AI·파티·성장 섹션

    아래 항목은 overview.md에 명확히 서술되어 있어 재정의 없이 참조 가능하다.

    카드 시스템 (2단계·3단계 관련)

    항목 정의 내용
    덱 영역 3종 대기덱·핸드덱·소모덱. 역할 각각 정의됨
    핸드 구성 전투 시작 시 대기덱에서 5장 드로우
    카드 발동 조건 액션 버튼 시 보드에 해당 숫자 타일이 있으면 발동. 타일 소모 후 효과 발동
    핸드 교체 규칙 액션 버튼 이후 핸드 전체(발동·미발동 모두)를 소모덱으로 이동 후 재드로우
    대기덱 소진 처리 소모덱 전체 셔플 → 대기덱 전환
    대기덱·소모덱 모두 0인 경우 드로우하지 않음. 핸드 빈 채로 진행
    사망 캐릭터 카드 처리 발동 불가 상태가 되며 핸드 교체 시 소모덱으로 이동
    중복 숫자 카드 선택 같은 숫자로 발동 가능한 카드가 여러 장이면 액션 버튼 전에 원하는 카드를 선택
    발동 숫자 등급표 4(기본)·8(일반)·16(강화)·32(강력)·64(필살)·128(초월)

    전투 시스템 (2단계·3단계·5단계 관련)

    항목 정의 내용
    출전 캐릭터 선택 전투 전 최대 3인 선택. 미출전 캐릭터 카드는 덱에서 제외
    턴 구조 순서 슬라이드(N회) → 액션 버튼 → 카드 실행 → 핸드 교체 → 적 카운트 처리
    전투 승리 조건 모든 적 처치 → 카드 보상(3장 중 1장) 획득 → 거점 귀환
    전투 패배 조건 파티 전원 사망 → 챕터 재시작 또는 세이브 포인트 복귀
    복수 적 동시 카운트 0 처리 적 목록 순서(위→아래)로 순차 실행
    카드 발동 중 적 사망 이미 선택한 카드의 효과는 완전 적용 후 전투 결과 판정

    적 AI 시스템 (4단계·5단계 관련)

    항목 정의 내용
    카운트 감소 타이밍 플레이어 액션 버튼 1회 → 모든 적 카운트 -1
    카운트 0 시 동작 예고 행동 실행 → 카운트 리셋
    예고 UI 표시 항목 행동 유형, 수치(피해량·효과량), 대상(단일·전체) 항상 공개
    적 유형 분류 일반·엘리트·보스 3종. 유형별 카운트 초기값 범위·행동 레퍼토리 다름
    보스 페이즈 전환 체력 구간마다 행동 패턴 교체. 카운트 리셋 + 새 예고 등록
    적 사망 시 카운트·예고 즉시 제거됨
    카운트 0 + 카드 효과로 동시 처치 해당 적의 행동 취소됨

    파티 연계 시스템 (3단계 관련)

    항목 정의 내용
    연계 조건 직전 카드와 현재 카드 소유 캐릭터가 다르고, 그 사이에 슬라이드 없어야 함
    연계 보너스 형태 추가 광역 피해 또는 버프 형태로 즉시 적용
    연계 타일 소모 각 카드의 발동 숫자 타일 2개 순서대로 소모. 두 타일 모두 보드에 존재해야 함
    이중 연계 세 캐릭터 연속 발동 시 보너스 중첩 가능

    2. 단계별 구현 항목과 문서화 상태

    2단계 — 카드·덱 시스템 (05/25–05/29)

    구현 항목 문서화 상태 비고
    카드 데이터 구조 (발동 숫자·효과 타입·소유 캐릭터) 미정의 개념 수준만 존재. 필드 목록·타입·제약 조건 없음
    대기덱·핸드·소모덱 3영역 관리 규칙 수준 정의 영역 역할은 정의됨. 영역 간 이동 처리 순서, 드로우 실패 처리 흐름 미정의
    전투 시작 시 5장 드로우 규칙 언급 덱이 5장 미만일 때 처리(소모덱 셔플 후 드로우 여부) 미정의
    액션 이후 핸드 전체 소모덱 이동 + 재드로우 규칙 수준 정의 재드로우 시 대기덱 부족 상황 처리 순서 미정의
    대기덱 소진 시 소모덱 셔플 → 대기덱 전환 규칙 언급 셔플 타이밍(드로우 직전 vs. 소진 감지 즉시) 미정의

    3단계 — 액션 버튼·카드 실행 (06/01–06/05)

    구현 항목 문서화 상태 비고
    액션 버튼 활성화 조건 암묵적 정의 "항상 누를 수 있다"는 맥락이지만 명시적 비활성화 조건(핸드 0장, 전투 종료 후 등) 없음
    핸드 카드 ↔ 보드 타일 매칭 로직 규칙 수준 정의 매칭 검사 순서, 타일 다수 존재 시 어떤 타일을 소모하는지 미정의
    매칭된 카드 효과 순차 실행 규칙 언급 카드 실행 순서(핸드 순서 vs. 발동 숫자 순서 vs. 매칭 완료 순서) 미정의
    미매칭 카드 자동 소모 처리 규칙 수준 정의 소모덱으로 이동 확인됨. 추가 처리 없음
    같은 숫자 카드 복수 존재 시 선택 UI 규칙 언급 선택 UI 발동 시점(액션 버튼 전 항상 vs. 중복 발견 시에만) 미정의
    카드 효과 유형 (공격·회복·버프·디버프) 미정의 효과 유형 목록, 각 유형의 파라미터 구조 없음
    파티 연계 발동 로직 규칙 수준 정의 연계 판정 시점(카드 선택 시 vs. 실행 시), 연계 보너스 수치 미정의

    4단계 — 적 AI (06/08–06/19)

    구현 항목 문서화 상태 비고
    적 데이터 구조 (체력·행동 카운트·행동 패턴) 미정의 필드 목록·타입 없음
    액션 버튼 1회 → 전체 적 카운트 -1 규칙 수준 정의 처리 순서 정의됨
    카운트 0 시 예고 행동 실행 → 카운트 리셋 규칙 수준 정의 카운트 리셋값(초기값 고정 vs. 패턴별 가변) 미정의
    행동 유형 구현 (단일 공격·광역·디버프·강화) 규칙 언급 각 행동 유형의 파라미터 구조, 적용 대상 선택 방식 미정의
    복수 적 동시 카운트 0 처리 규칙 수준 정의 위→아래 순서 정의됨
    보스 페이즈 전환 규칙 언급 페이즈 전환 체력 구간 정의 방식(비율 vs. 절대값), 페이즈 전환 연출 규칙 미정의
    같은 턴 여러 스킬 겹침 처리 미결 상태 overview.md에 "추후 결정"으로 명시된 미해결 항목

    5단계 — 전투 UI (06/22–07/03)

    구현 항목 문서화 상태 비고
    전투 화면 레이아웃 미정의 ux.md 전체가 플레이스홀더
    보드 슬라이드 애니메이션·타일 병합 이펙트 미정의 연출 규칙, 연출 중 입력 처리 방식 없음
    핸드 카드 표시·매칭 타일 하이라이트 미정의 하이라이트 조건, 실시간 반영 여부 없음
    적 행동 예고 UI 규칙 수준 정의 표시 항목은 정의됨. UI 배치·카운트 감소 애니메이션 규칙 미정의
    슬라이드 잔여 횟수 표시 미정의 표시 위치, 소진 시 UI 상태 없음
    카드 실행 연출 (공격 이펙트·피해 수치 팝업) 미정의 연출 중 다음 입력 처리 방식, 연출 스킵 여부 없음

    3. 단계별 미결 사항

    3-A. 2단계 미결 사항 — 카드·덱 시스템

    # 미결 항목 선택지 영향 범위
    A-1 카드 효과 타입 목록 확정 (a) 공격·회복·버프·디버프 4종으로 고정 / (b) 추가 타입(실드·드로우·상태이상 등) 포함한 확장 목록 카드 데이터 구조 설계에 직결. 이후 모든 카드 데이터 작성의 기반
    A-2 전투 시작 드로우 시 대기덱 5장 미만 처리 (a) 있는 장수만 드로우하고 소모덱 셔플 후 나머지 드로우 / (b) 있는 장수만 드로우하고 셔플 없이 시작 전투 시작 초기화 로직 분기 발생
    A-3 셔플 타이밍 (a) 대기덱이 0이 되는 순간 즉시 셔플 / (b) 드로우 시도 시점에 대기덱이 0이면 그때 셔플 타이밍 차이가 드로우 결과에 영향. (b)가 표준적
    A-4 카드 강화(합성) 결과 정의 (a) 발동 숫자만 감소 (예: 16→12) / (b) 발동 숫자 감소 + 효과 강화 중 하나 선택 / (c) 카드마다 강화 결과가 고정 카드 데이터 구조에 "강화 단계" 필드 필요 여부 결정
    A-5 덱 구성 상한 카드 상한 없음 vs. 최대 N장 제한 밸런스 및 소모덱 순환 주기에 영향

    3-B. 3단계 미결 사항 — 액션 버튼·카드 실행

    # 미결 항목 선택지 영향 범위
    B-1 카드 실행 순서 (a) 핸드에서 왼쪽(낮은 인덱스) 순서 / (b) 발동 숫자 오름차순 / (c) 발동 숫자 내림차순 / (d) 플레이어가 순서 지정 연계 발동 판정, 효과 적용 순서(방어막 선적용 여부 등)에 직결
    B-2 동일 숫자 타일 다수 존재 시 소모 위치 선택 (a) 자동 선택(무작위 또는 위치 규칙 고정) / (b) 플레이어가 소모할 타일 위치를 직접 선택 UX 복잡도와 전략성에 영향. (b)는 추가 선택 UI 필요
    B-3 중복 숫자 카드 선택 UI 발동 시점 (a) 액션 버튼을 누르기 전에 핸드에서 항상 선택 가능 / (b) 액션 버튼 누른 뒤 매칭 발견 시 팝업 (a)는 사전 계획 강조, (b)는 확인 절차 강조. 구현 방식 차이 큼
    B-4 액션 버튼 비활성화 조건 명시 항상 활성 vs. 핸드 0장일 때 비활성 vs. 매칭 카드 0장일 때 비활성 UI 상태 관리와 플레이어 혼란 방지 설계에 영향
    B-5 연계 보너스 수치 정의 추가 광역 피해의 기본값, 계산 기준(카드 기본 피해의 %, 고정값 등) 밸런스에 직결. 패시브 "연계 전문" +50% 적용 기준도 여기서 결정
    B-6 연계 판정 시점 (a) 카드 실행 처리 시 직전 카드와 비교 / (b) 플레이어가 카드 순서를 지정한 시점에 미리 판정 B-1(카드 실행 순서)과 연동됨

    3-C. 4단계 미결 사항 — 적 AI

    # 미결 항목 선택지 영향 범위
    C-1 카운트 리셋값 방식 (a) 적마다 고정된 초기값으로 항상 리셋 / (b) 행동 패턴별로 다른 리셋값 사용 / (c) 리셋값에 범위(최소~최대) 적용, 무작위 선택 적 행동의 예측 가능성과 전략 깊이에 직결
    C-2 행동 패턴 구조 (a) 고정 순환(1→2→3→1) / (b) 랜덤 선택(가중치 풀) / (c) 조건 분기(체력 구간·카운트에 따라 패턴 선택) 적 데이터 구조의 복잡도 결정. 보스에서 특히 중요
    C-3 같은 턴 여러 스킬 겹침 처리 (overview.md에 "추후 결정"으로 명시된 항목) (a) 등록 순서대로 모두 실행 / (b) 첫 번째 스킬만 실행, 나머지 무시 / (c) 합산하여 하나의 행동으로 처리 복잡한 보스 기믹 설계 가능 여부에 영향. 구현 복잡도 차이 큼
    C-4 보스 페이즈 전환 트리거 정의 (a) 체력 비율(50%, 25% 등) / (b) 절대 체력값 / (c) 특정 카운트 도달 시 페이즈 전환 체력값이 보스 데이터에서 어떻게 표현되는지 결정
    C-5 디버프·버프의 지속 방식 (a) N턴 지속 (액션 버튼 횟수 기준) / (b) 영구 지속(해제 스킬 없으면 유지) / (c) 조건부 해제 상태이상 시스템 전반 설계에 영향. 카드 효과 타입(A-1)과 연동
    C-6 적 행동 대상 선택 방식 (단일 공격 시) (a) 항상 첫 번째 생존 파티원 / (b) 체력이 가장 낮은 파티원 / (c) 무작위 파티원 / (d) 적 패턴 데이터에 타겟팅 방식을 명시 전략성 수준과 플레이어 대응 가능성에 영향

    3-D. 5단계 미결 사항 — 전투 UI

    # 미결 항목 선택지 영향 범위
    D-1 연출 중 입력 처리 방식 (a) 연출 중 모든 입력 차단 (연출 완료 후 입력 수신) / (b) 연출 스킵 가능 (탭으로 스킵) / (c) 연출과 무관하게 즉시 입력 수신 UX 체감과 게임 속도에 큰 영향. 카드 실행·보드 슬라이드 애니메이션 모두 적용
    D-2 매칭 타일 하이라이트 실시간 반영 여부 (a) 항상 실시간 (보드 상태 변경될 때마다 갱신) / (b) 액션 버튼 누르기 직전에만 표시 구현 비용과 플레이어 가독성에 영향
    D-3 전투 화면 레이아웃 기본 구조 결정 보드·핸드·적 목록의 배치 방향 및 비율 (예: 상단 적 / 중단 보드 / 하단 핸드 vs. 좌측 보드 / 우측 적·핸드) 5단계 UI 구현 전체의 기반. 미결 시 UI 작업 착수 불가

   - DeckState 구조체 (대기덱·핸드덱·소모덱 각 영역 배열)
   - 영역 간 이동 규칙 상세
   - 전투 시작 초기화 순서 (셔플 → 드로우 5장, 부족 시 처리 — 미결 A-2 확정 후)

   - 드로우 트리거 2종: 전투 시작, 액션 버튼 이후
   - 대기덱 소진 시 셔플 타이밍 (미결 A-3 확정 후)

4. 카드 발동 흐름
   - 매칭 검사 순서
   - 타일 소모 처리
   - 효과 실행 순서 (미결 B-1 확정 후)
   - 미매칭 카드 처리

5. 카드 강화 (합성)
   - 강화 조건: 같은 카드 2장
   - 강화 결과 (미결 A-4 확정 후)

6. 카드 보상
   - 전투 종료 후 3장 중 1장 선택 흐름
   - 챕터 클리어 시 초기화 처리

   - 슬라이드 처리 후 상태 갱신 순서

     (2) 핸드·보드 매칭 검사
     (3) 카드 효과 실행 (순서 — 미결 B-1 확정 후)
     (4) 파티 연계 판정 및 보너스 적용
     (5) 핸드 전체 소모덱 이동
     (6) 재드로우
     (7) slideCount 리셋
