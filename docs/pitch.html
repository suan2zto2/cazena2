Core Concept

핵심 컨셉

제한된 슬라이드 안에서, 핸드를 얼마나 완성할 것인가.
매 턴 핸드의 카드마다 발동에 필요한 숫자가 정해져 있다. 보드 슬라이드로 타일을 병합해 그 숫자를 만들어야 카드가 발동된다.

핸드 드로우

매 턴 5장. 각 카드에 4·8·16·32·64·128 중 하나의 발동 숫자가 지정됨.

보드 조작

4×4 격자를 슬라이드해 타일을 병합. 턴당 슬라이드 횟수 제한이 핵심 자원.

액션 버튼

보드 타일을 행동 재화로 변환. 발동 가능한 카드만 실행되고, 핸드는 전체 교체.

Core Loop

한 턴의 흐름

1 핸드덱에서 카드 5장 드로우 — 각 카드의 필요 숫자 확인
↓
2 보드 슬라이드 (제한 횟수 내) — 카드에 맞는 타일 준비
↓
3 액션 버튼 — 타일 → 행동 재화 변환, 매칭 카드 일괄 발동
↓
4 핸드 전체 교체, 모든 적 행동 카운트 -1
↓
5 카운트 0인 적 행동 실행 → 다음 턴 반복
Combat System · 01

보드 시스템 — 2048 에너지 보드

4×4 타일 격자. 상·하·좌·우로 슬라이드하면 같은 숫자 타일이 병합되어 두 배 값으로 자란다. 높은 숫자일수록 강력한 카드를 발동할 수 있지만, 만들려면 슬라이드를 더 많이 소모한다.

슬라이드 규칙

  • 인접한 동일 숫자 타일은 병합 → 두 배 값
  • 슬라이드 1회마다 빈 칸에 2 또는 4 생성
  • 턴당 슬라이드 횟수 제한 — 자원 압박의 원천
  • 액션 버튼 시 보드 타일 전체가 행동 재화로 변환
Combat System · 02

카드 시스템 — 숫자가 곧 위력

각 카드는 발동에 필요한 숫자 타일이 지정되어 있다. 높은 숫자일수록 효과가 강력하지만, 만들기 어렵다. 3인 파티의 카드를 합산한 공용 덱에서 매 턴 5장을 드로우한다.

덱 영역

대기덱 · 핸드덱(5장) · 소모덱 세 영역을 순환한다.

강화

같은 카드 2장을 합성하면 발동 숫자가 한 단계 낮아진다 — 더 적은 자원으로 같은 효과.

보상

전투 승리 시 카드 3장 중 1장을 선택해 덱에 추가한다.

Combat System · 03

적 AI — 완전 공개형 카운트다운

적은 다음에 할 행동을 항상 공개한다. 액션 버튼 1회당 카운트 -1, 0이 되면 예고된 행동이 즉시 실행된다. 정보는 모두 보이지만, 슬라이드와 핸드의 제약 때문에 완벽한 대응이 어렵다.

Why It Matters

매 턴의 딜레마

슬라이드는 유한하고, 적은 한 턴씩 다가오고, 핸드는 도박이다.

"핸드에 64짜리 필살 카드가 있다.
만들려면 슬라이드가 5번은 더 필요한데 남은 횟수가 3번밖에 없다.
64를 포기하고 4·8·16만 확실히 실행할까,
아니면 도박을 걸까?"
Combat System · 04

파티 연계 — 3인의 협동

슬라이드 없이 서로 다른 캐릭터의 카드를 연속 발동하면 연계 보너스가 추가로 터진다. 단, 타일 2개를 동시에 소모하므로 보드 계획이 선행되어야 한다.

연계 조건

  • 직전 카드와 현재 카드의 소유자가 달라야 함
  • 그 사이에 슬라이드 입력이 없어야 함
  • 두 카드의 타일이 보드에 모두 존재해야 함

보너스 효과

  • 추가 광역 피해 또는 버프 즉시 적용
  • 3인 연속 발동 시 이중 연계로 중첩
  • 패시브 "연계 전문" 보유 시 +50%
