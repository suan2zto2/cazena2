## 핵심 컨셉

**슬라이드 한 번 한 번이 전략적 결정이다.**

- 보드를 슬라이드할 때마다 **행동 카운트**가 소모됨
- 적은 각자 **행동 카운트**를 가지며, 카운트가 0이 되면 공격 실행
- 플레이어는 손에 든 스킬 카드를 보유하고, 카드마다 **발동에 필요한 숫자 타일**이 지정됨
- 보드에서 해당 숫자 타일을 완성하면 카드를 발동할 수 있음
- **높은 숫자일수록 강력한 카드** — 그러나 높은 숫자를 만들려면 슬라이드가 더 필요

핵심 딜레마:
> *"적이 다음 턴에 강력한 공격을 예고했다. 내 손에는 32짜리 방어 카드가 있는데, 보드에는 아직 16밖에 없다. 한 번 더 슬라이드해서 32를 만들까? 아니면 지금 있는 16짜리 카드를 써야 할까?"*

## 코어 루프

```
스토리 챕터 진행
    ↓
전투 돌입 — 보드 초기화 + 손패 드로우
    ↓
보드 슬라이드 (행동 카운트 소모)
    ↓
목표 숫자 타일 완성 → 카드 발동
    ↓
적 카운트 0 도달 → 적 행동 실행
    ↓
[반복] 적 전멸 → 전투 승리 → 카드 보상 획득
    ↓
거점 귀환 → 덱 편집 / 캐릭터 강화
    ↓
다음 챕터 진입
```

## 전투 시스템

### 보드 슬라이드와 행동 카운트

```
[ 에너지 보드 ]          [ 적 행동 카운트 ]

┌────┬────┬────┬────┐    고블린 검사   ▶ 3
│  2 │    │  4 │    │    오크 마법사   ▶ 1  ← 다음 턴 공격!
├────┼────┼────┼────┤    보스 드래곤   ▶ 5
│    │  2 │    │  4 │
├────┼────┼────┼────┤
│  4 │    │  2 │    │
├────┼────┼────┼────┤
│    │  4 │    │  2 │
└────┴────┴────┴────┘
```

- 플레이어가 보드를 **1회 슬라이드**할 때마다 **모든 적의 카운트가 1 감소**
- 카운트가 0인 적은 **즉시 예고된 행동 실행** 후 카운트 리셋
- 카드 발동은 카운트 소모 없음 — 타일이 준비된 순간 언제든 발동 가능

### 스킬 카드와 발동 조건

- 플레이어는 3인 파티 전체의 스킬 카드를 섞은 덱에서 매 전투 초기 **5장 드로우**
- 각 카드에는 **발동 숫자**가 표시됨

| 발동 숫자 | 스킬 등급 | 예시 |
|:---------:|:---------:|------|
| 4 | 기본 | 단일 약공격, 소량 방어막 |
| 8 | 일반 | 단일 공격, 디버프 1개 |
| 16 | 강화 | 중간 공격, 광역 디버프, 아군 회복 |
| 32 | 강력 | 강한 광역 공격, 파티 전체 방어막 |
| 64 | 필살 | 캐릭터 전용 필살기 |
| 128 | 초월 | 전투 판도를 뒤집는 초강력 효과 |

- 보드에 해당 숫자 타일이 존재하면 **타일 소모 → 카드 발동**
- 같은 숫자로 발동 가능한 카드가 여러 장이면 원하는 카드를 선택

### 적 행동 예고

- 모든 적은 항상 **다음 행동**을 공개 표시
  - 예: "오크 마법사 — 카운트 1 — 전체 공격 12 예고"
- 플레이어는 이 예고를 보고 남은 슬라이드 횟수와 타일 상황을 계산해 우선순위 결정

### 핵심 긴장 구조

```
슬라이드를 많이 할수록
    → 높은 숫자 타일 완성 가능 → 강력한 카드 발동
    → 그러나 적의 카운트가 빠르게 줄어듦 → 적 행동 더 빨리 옴

슬라이드를 아낄수록
    → 적 행동을 늦출 수 있음
    → 그러나 낮은 숫자 카드만 쓸 수 있음 → 전투가 길어짐
```

이 딜레마가 매 전투, 매 슬라이드마다 반복됨

### 보드 위기 상황

- 보드가 꽉 차면 슬라이드 시 새 타일이 생성되지 않음
- 슬라이드해도 타일이 이동하지 않으면 **행동 불능** — 카드 발동도 슬라이드도 막힘
- 보드 관리 실패 = 전략 붕괴의 직접 원인

### 파티 연계 발동

- 같은 턴(슬라이드 없이 연속)에 서로 다른 캐릭터의 카드를 연속 발동하면 **연계 보너스**
  - 예: A 캐릭터 16 카드 → B 캐릭터 16 카드 연속 발동 → 추가 광역 피해
- 연계는 타일 2개를 한 번에 소모하므로 보드 계획이 필요

## 성장 시스템

### 덱 성장
- 전투 보상: 카드 3장 중 1장 선택 → 덱에 추가
- 카드 강화: 같은 카드 2장 합성 → 발동 숫자 감소 (예: 16 → 12) 또는 효과 강화
- 덱은 리셋 없이 게임 전체 유지

### 캐릭터 레벨 패시브
- 레벨업 시 보드 / 전투 관련 패시브 선택

| 패시브 예시 | 효과 |
|-------------|------|
| 선제 타일 | 전투 시작 시 보드에 4 타일 2개 추가 생성 |
| 타일 절약 | 카드 발동 후 소모 타일 수치의 절반을 보드에 반환 |
| 카운트 지연 | 전투 시작 시 모든 적 카운트 +1 |
| 연계 전문 | 연계 발동 보너스 피해 +50% |

### 유물
- 챕터 보스 처치 시 유물 획득 → 보드 규칙 또는 카드 조건을 영구 변경

| 유물 예시 | 효과 |
|-----------|------|
| 시간의 모래 | 특정 카드 발동 시 모든 적 카운트 +1 |
| 황금 나침반 | 슬라이드 3회마다 원하는 위치에 타일 1개 지정 생성 |
| 공명석 | 64 이상 타일 소모 시 추가로 손패 1장 드로우 |

## 패키지 게임 특성 반영

| 요소 | 설계 방향 |
|------|-----------|
| 엔딩 구조 | 챕터 클리어 누적 → 최종 챕터 보스 격파 시 엔딩. 수집 요소에 따라 엔딩 분기 |
| 학습 곡선 | 초반은 낮은 카운트 적 + 낮은 숫자 카드 위주 → 후반 챕터에서 짧은 카운트 적 + 높은 숫자 카드 요구 |
| 전략 깊이 | 슬라이드 횟수 계산 + 손패 우선순위 + 적 행동 예고 대응이 동시에 맞물림 |
| 재플레이 유인 | 덱 방향 (저숫자 다발 vs. 고숫자 필살) + 패시브 선택에 따라 완전히 다른 플레이 스타일 |
