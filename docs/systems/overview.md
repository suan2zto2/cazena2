## 캐릭터

### 규칙 및 동작 방식

- 플레이어는 기본 제공 캐릭터를 포함하여 캐릭터를 보유하며, 전투 시작 시 그 중 1인을 선택하여 출전시킨다. 기본 제공은 최대 5인으로 예상하며, DLC 등을 통해 추가될 수 있다.
- 선택한 캐릭터의 카드 덱이 해당 전투의 덱을 구성한다.
- 출전하지 않은 캐릭터의 카드는 해당 전투 덱에 포함되지 않는다.
- 캐릭터 체력이 0이 되면 전투 패배로 판정한다.

## 보드 시스템

### 개요

보드 시스템은 2048 규칙을 기반으로 한 4×4 타일 격자다. 플레이어는 상·하·좌·우 중 한 방향으로 보드를 슬라이드하여 같은 숫자 타일을 병합하고, 핸드 카드 발동에 필요한 숫자 타일을 준비한다. 턴당 슬라이드 횟수가 제한되어 있으므로, 제한된 횟수 안에서 어떤 타일 조합을 만들지가 핵심 결정이 된다.

### 규칙 및 동작 방식

- 보드 기본 크기는 4×4(16칸)
- 슬라이드 시 이동 방향으로 모든 타일이 밀리고, 인접한 동일 숫자 타일은 병합되어 두 배 값의 타일이 된다.
- 슬라이드 1회 후 빈 칸 중 무작위 1칸에 값 2 또는 4의 새 타일이 생성된다.
- 턴당 슬라이드 횟수에 제한이 있으며, 제한 내에서 원하는 타일 조합을 만드는 것이 목표다.
- 액션 버튼을 누르면 보드 위의 모든 숫자 타일이 행동 재화로 변환된다. 이후 핸드의 카드 중 해당 재화로 발동 가능한 카드가 실행되며, 실행된 만큼 재화가 소모된다.
- 타일이 카드 발동 숫자와 정확히 일치해야 소모 가능

### 예외 처리

- 슬라이드해도 어떤 타일도 이동하지 않으면 **슬라이드 불능** 상태가 되어 슬라이드 입력만 차단된다. 액션 버튼은 정상적으로 누를 수 있다.
- 슬라이드 방향에 병합 가능한 타일이 없더라도 타일이 물리적으로 이동했다면 유효한 슬라이드로 간주하여 카운트를 소모하고 새 타일을 생성한다.

## 전투 시스템

### 개요

![전투 화면](../assets/images/combat_overview.png)

전투는 보드 초기화 및 핸드덱 드로우로 시작하여 "슬라이드(N회 이내) → 액션 버튼 → 카드 실행 → 핸드 교체 → 적 행동"의 사이클을 적이 전멸할 때까지 반복하는 구조다. 제한된 슬라이드 횟수 안에서 핸드의 카드 숫자를 보드에 얼마나 준비할 수 있는지가 핵심이며, 높은 숫자를 욕심내다 슬라이드를 소진하면 그 턴 실행되는 카드가 줄어드는 리스크가 생긴다.

### 규칙 및 동작 방식

- 전투 시작 전 보유 캐릭터 중 출전할 캐릭터를 선택한다(1인). 출전하지 않는 캐릭터의 카드는 이번 전투 덱에 포함되지 않는다.
- 전투 시작 시 보드는 초기화되고 대기덱에서 5장을 드로우하여 핸드덱을 구성한다.
- 플레이어는 턴당 정해진 횟수 이내에서 보드를 슬라이드하여 핸드 카드에 맞는 타일을 준비한다. 슬라이드는 적 행동 카운트에 영향을 주지 않는다.
- 액션 버튼을 누르면 보드 위의 모든 타일이 행동 재화로 변환된다. 이후 핸드의 카드 중 해당 재화로 발동 가능한 카드가 실행되며, 재화가 없는 카드는 실행되지 않는다.
- 액션 버튼 이후 핸드 전체를 버리고 대기덱에서 새로 드로우하여 핸드덱을 재구성한다.
- 카드 실행 및 핸드 교체가 끝나면 전장의 모든 적 행동 카운트가 1 감소하며, 카운트가 0이 된 적은 즉시 예고 행동을 실행한 뒤 카운트를 리셋한다.
- 모든 적을 처치하면 전투 승리 — 카드 보상(3장 중 택 1)과 골드를 획득하고 다음 스테이지 선택 화면으로 이동한다.
- 캐릭터 체력이 0이 되면 전투 패배 — 챕터 진행이 종료되고 챕터 내 획득 정보가 소멸된다. 진행한 스테이지 수에 비례한 경험치만 지급된다.

### 예외 처리

- 같은 액션 버튼으로 여러 적의 카운트가 동시에 0이 되면, 적 목록 순서대로(위→아래) 행동을 순차 실행한다.
- 카드 발동 도중 적이 사망해도 이미 선택한 카드의 효과는 완전히 적용된 뒤 전투 결과를 판정한다.

## 카드 시스템

### 개요

카드 시스템은 선택 캐릭터의 스킬 카드를 대기덱·핸드덱·소모덱 세 영역으로 통합 관리하는 구조다. 각 카드에는 발동에 필요한 숫자(4·8·16·32·64·128)가 지정되어 있으며, 숫자가 높을수록 강력한 효과를 제공한다. 전투 보상을 통해 챕터 내에서 덱이 성장하며, 챕터 클리어 시 초기화된다.

### 덱 운영 구조

카드는 세 영역으로 운영된다.

| 영역 | 설명 |
|------|------|
| 대기덱 | 아직 드로우되지 않은 카드. 셔플된 상태로 쌓여 있음 |
| 핸드덱 | 현재 사용 가능한 카드 (항상 5장 유지) |
| 소모덱 | 발동·버리기로 사용된 카드 |

### 규칙 및 동작 방식

- 선택 캐릭터의 카드를 셔플하여 대기덱으로 구성하고, 전투 시작 시 5장을 드로우하여 핸드덱을 구성한다.
- 카드 발동 조건: 액션 버튼을 눌렀을 때 보드 타일이 변환된 행동 재화 중 해당 발동 숫자에 맞는 재화가 존재할 것. 조건을 충족한 카드는 재화를 소모하고 효과가 즉시 발동된다.
- 액션 버튼 이후 핸드의 모든 카드(실행된 카드, 실행되지 않은 카드 모두)가 소모덱으로 이동하고, 대기덱에서 새로 드로우하여 핸드덱을 재구성한다.
- 대기덱이 0장이 되면 소모덱 전체를 셔플하여 대기덱으로 이동한다.
- 같은 숫자로 발동 가능한 카드가 핸드덱에 여러 장이면 액션 버튼을 누르기 전 원하는 카드를 선택할 수 있다.
- 전투 보상(3장 중 1장 선택)을 덱에 추가한다. 챕터 종료 시 함께 소멸된다.

### 예외 처리

- 대기덱이 비어 있을 때 소모덱도 0장이면 드로우하지 않는다. 핸드덱이 빈 채로 다음 슬라이드 단계를 진행한다.
- 유물·패시브 등 특정 옵션에 의해 발동 숫자와 다른 재화로 카드를 발동하는 것도 가능하다.

## 적 AI 시스템

### 개요

적 AI 시스템은 행동 카운트 기반의 예측 가능한 구조를 사용한다. 각 적은 초기 카운트 값을 가지며 플레이어가 액션 버튼을 누를 때마다 1씩 감소하고, 0이 되는 순간 사전에 공개된 행동을 실행한다. 플레이어는 적의 다음 행동과 카운트를 항상 볼 수 있으므로, 전략적 대응이 가능하다.

### 규칙 및 동작 방식

- 적 행동 예고 UI에는 행동 유형(공격·디버프·강화 등), 수치(피해량·효과량), 대상(단일·전체)이 항상 표시된다.
- 플레이어의 액션 버튼 1회 → 모든 적 카운트 -1 → 카운트 0인 적은 예고 행동 실행 → 카운트 리셋.
- 각 적에게는 턴 수마다 발동할 스킬이 등록되어 있으며, 해당 턴에 발동할 스킬이 없으면 일반 공격이 발동된다.
- 적은 일반·엘리트·보스 세 유형으로 분류되며, 유형에 따라 카운트 초기값 범위와 행동 레퍼토리가 다르다.
- 보스는 체력 구간(페이즈)마다 행동 패턴이 교체되어 카운트 리셋과 함께 새 예고가 등록된다.
- 카드 발동으로 적을 처치하면 해당 적의 카운트 및 예고 행동이 즉시 제거된다.

### 예외 처리

- 적의 공격 피해는 방어막이 있을 경우 방어막을 먼저 소모하고, 남은 피해를 캐릭터 체력에서 차감한다.
- 카운트가 0인 적이 카드 효과로 동시에 처치되면 해당 적의 행동은 실행되지 않고 취소된다.
- 같은 턴에 발동할 스킬이 여러 개 겹치는 경우의 처리 규칙은 추후 결정.
