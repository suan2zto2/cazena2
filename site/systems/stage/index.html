    스테이지 진행 시스템

    📄 docs/systems/stage.md

    개요

    게임의 챕터 진행 구조를 정의한다. 메인 스토리, 서브 스토리, 도전 로그라이크 모드가 모두 동일한 챕터 구조를 사용한다.

    구분 메인 스토리 서브 스토리 도전 로그라이크
    챕터 수 8챕터 (순차 해금) 스토리마다 1챕터 (조건부 해금) 1챕터 (반복 도전)
    해금 방식 이전 챕터 클리어 메인 스토리 진행 중 특정 조건 달성 별도 모드 진입
    클리어 후 처리 다음 챕터 해금 + 경험치 경험치 + 스토리 완결 팀덱 저장 후 계속 도전

    로그라이크 스테이지 유형

    챕터 로그라이크 구조는 사용되는 게임 모드에 따라 두 가지 유형으로 동작한다.

    유형 클리어 후 처리 사용 모드
    일반 로그라이크 스테이지 팀덱 저장 여부 확인 후 스테이지 종료 스토리 모드 (챕터당 1개 배치)
    도전 로그라이크 스테이지 팀덱 저장 여부 확인 후 추가 도전 계속 진행 도전 로그라이크 모드

    일반 로그라이크 스테이지

    스토리 모드에서 챕터당 1개 배치된다.

    • 클리어 후 팀덱 저장 여부를 확인하고 스테이지가 종료된다.
    • 스토리 진행 몰입을 위해 클리어 후 반복 도전을 허용하지 않는다.
    • 저장 가능 시점 기준은 미정 (예: 3막 이상 진행 완료).
    • 저장 결과: 일반 팀덱으로 보관.

    도전 로그라이크 스테이지

    도전 로그라이크 모드 전용.

    • 챕터 클리어 후에도 계속 도전할 수 있으며 등장 몬스터가 점점 강해진다.
    • 챕터 클리어 시점마다 일반 팀덱으로 저장할 수 있다.
    • 최종 패배 시 해당 시점의 팀 구성을 도전 팀덱으로 저장할 수 있다.

    챕터 구조

    로그라이크 진행 방식

    챕터 진행 중 획득한 아이템(스킬카드, 유물, 패시브 등)은 해당 챕터 내에서만 유효하다.

    챕터 입장
    ↓
스테이지 진행 → 스킬카드·유물·패시브 획득 (챕터 내 누적)
    ↓
챕터 종료 (클리어 또는 실패)
    ↓
챕터 내 습득 정보 전부 소멸
    ↓
[클리어 시] 챕터 클리어 기록 영구 저장 → 다음 챕터 해금
[실패 시]  진행한 스테이지 수에 비례한 경험치만 획득

    챕터 내 아이템은 다른 챕터에서 사용할 수 없으며, 챕터가 종료되는 순간 모두 소멸된다. 챕터 클리어 여부만 영구 기록으로 남는다.

    막(Act) 구조

    챕터는 여러 개의 막(Act)으로 구성된다. 기본은 3막이며, 챕터 테이블 설정에 따라 줄거나 늘어날 수 있다.

    챕터
├── 1막
│   ├── 스테이지 라인 (유저 경로 선택)
│   └── 중간 보스 → 클리어 시 2막 진입
├── 2막
│   ├── 스테이지 라인 (유저 경로 선택)
│   └── 중간 보스 → 클리어 시 3막 진입
└── 3막
    ├── 스테이지 라인 (유저 경로 선택)
    └── 최종 보스 → 클리어 시 챕터 완료
    요소 설명
    막 수 챕터 테이블에서 설정 (기본 3막)
    막당 스테이지 수 챕터 테이블에서 설정
    마지막 스테이지 항상 보스 고정
    보스 종류 마지막 막 = 최종 보스 / 나머지 막 = 중간 보스

    스테이지 라인 (경로 선택)

    각 막의 스테이지는 슬레이 더 스파이어·카오스 제로 나이트메어와 같은 방식으로 배치된다.

    • 시작 지점에서 보스 지점까지 랜덤으로 펼쳐진 경로가 생성된다.
    • 유저는 다음 스테이지를 여러 후보 중에서 선택하며 진행한다.
    • 보급 지역의 위치는 챕터 테이블에서 지정된 위치에 고정 배치된다.
    • 보스 지점은 항상 경로의 마지막에 존재한다.
    시작
 ├─[전투]─[엘리트]─[보급]─┐
 │                         ├─[전투]─[보스]
 └─[전투]─[미확인]─[전투]─┘

    스테이지 종류

    일반 전투 (Normal)

    기본 전투 스테이지. 챕터 초반과 중반에 다수 배치된다.

    항목 내용
    내용 일반 몬스터와 전투
    주요 보상 스킬카드 3장 중 1장 선택 획득
    기본 보상 골드 획득

    엘리트 전투 (Elite)

    일반보다 강한 몬스터가 등장하는 고위험·고보상 스테이지.

    항목 내용
    내용 엘리트 몬스터와 전투
    주요 보상 스킬카드 3장 중 1장 선택 획득 (일반 대비 고품질)
    기본 보상 골드 획득 (일반 대비 증가)

    보스 (Boss)

    막의 마지막 지점. 반드시 통과해야 다음 막으로 진행할 수 있다.

    종류 조건 특이사항
    중간 보스 마지막 막 이전의 보스 클리어 시 다음 막 진입
    최종 보스 마지막 막의 보스 클리어 시 챕터 클리어

    보스 클리어 시 유물 1개를 획득한다.

    보급 지역 (Supply)

    전투 없이 세 가지 행동 중 하나를 선택할 수 있는 휴식 지점.

    선택지 내용
    체력 회복 파티 캐릭터들의 체력을 일정량 회복
    상점 이용 골드로 스킬카드·유물 구매 가능
    카드 강화 보유한 스킬카드 1장의 등급 업그레이드

    세 가지 중 하나만 선택 가능하다.

    미확인 지역 (Unknown)

    입장 전에 내용을 알 수 없는 이벤트 지역. 다양한 상황이 랜덤으로 발생한다.

    발생 유형 내용
    아이템 획득 선택지에 따라 골드·카드·유물 획득
    전투 예상치 못한 적과 조우. 승리 시 추가 보상
    유물 획득 유물 1개를 즉시 획득 또는 조건부 획득

    보통 무언가 보상을 얻게 되지만, 선택에 따라 불이익이 발생하는 경우도 있다.

    스테이지 배치 구성

    챕터 테이블과 막 테이블에서 각 막의 구성을 설정한다.

    설정 항목 테이블 내용
    actCount Chapter 챕터의 막 수
    stageCount ActConfig 막당 비보스 스테이지 수
    supplyPositions ActConfig 보급 지역이 고정 배치될 위치 번호 (1-based)
    bossMonsterId ActConfig 막 보스로 사용할 몬스터 ID

    막별 스테이지 패턴

    스테이지 타입은 막 위치에 따라 고정 패턴으로 배치된다. 랜덤 요소는 어느 경로를 선택하느냐이며, 각 위치의 타입 자체는 사전에 결정된다.

    막 위치 1 위치 2 위치 3 위치 4 위치 5
    1·2막 NORMAL NORMAL ELITE SUPPLY BOSS
    3막 (최종) NORMAL ELITE ELITE UNKNOWN BOSS

    스토리 씬

    챕터 진행 중 특정 시점에 스토리 씬(컷씬·대화)을 삽입할 수 있다.

    기본 발동 시점

    트리거 설명
    CHAPTER_START 챕터 진입 직후. 해당 챕터의 도입부
    BOSS_START 보스 전투 시작 직전. 중간보스·최종보스 모두 해당
    BOSS_CLEAR 보스 처치 직후. 클리어 연출·다음 전개

    보스 씬은 중간보스와 최종보스에 각각 독립적으로 등록할 수 있다.

    중간 스테이지 씬

    기능상 일반·엘리트·보급·미확인 스테이지에도 씬을 등록할 수 있다(STAGE_START 트리거). 단, 챕터 진행이 경로 선택 방식이므로 플레이어가 해당 스테이지를 방문하지 않을 수 있다. 따라서:

    • 필수 스토리 라인(주요 서사·복선·캐릭터 설정)은 중간 스테이지에 배치하지 않는다.
    • 방문해도 그만, 안 해도 그만인 부가적 연출에만 사용한다.

    스키마 연결

    스토리 씬은 StoryScene 기획 테이블에 등록한다. 각 씬은 어느 챕터·어느 시점에 발동할지를 트리거로 지정한다. 상세 구조는 스키마 설계 → StoryScene 참조.
